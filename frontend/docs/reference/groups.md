# 分组说明

分组决定这把 Key 能走哪类协议、能看到哪些模型、会按照什么规则计费。把分组理解清楚，能省掉大部分接入和排障时间。

::: important 先用一句话记住分组
分组决定你这把 Key 走哪个平台、看得到哪些模型、按什么规则计费，以及是否存在额度或专属限制。
:::

## 先理解分组的作用

一把 Key 绑定到某个分组后，后续请求会受到这个分组影响：

- 走哪个平台协议
- 能看到哪些模型
- 是否允许混合调度
- 是标准计费还是订阅模式
- 是否有日 / 周 / 月额度限制

::: warning 没有分组时，请求不会被正常转发
后端会对未分组 Key 做拦截，所以只创建 Key 而不选分组，实际是不能用的。
:::

## 分组里最值得看哪些字段

| 字段 | 含义 |
| --- | --- |
| `platform` | 平台类型，决定主要协议入口 |
| `rate_multiplier` | 计费倍率 |
| `subscription_type` | 标准计费 or 订阅模式 |
| `is_exclusive` | 是否专属分组 |
| `daily / weekly / monthly limit` | 额度上限 |
| `fallback_group_id` | 失败时的回退分组 |

::: tip 想把价格看明白时，再继续读计费页
如果你已经知道分组会影响价格，但还不清楚“刀”“汇率”“输入价格”“缓存价格”之间是什么关系，继续看 [计费与价格说明](./pricing.md) 会更直接。
:::

## 常见平台类型

### `anthropic`

适合 Claude / Anthropic 兼容场景，通常配合：

- `/v1/messages`
- `/v1/messages/count_tokens`
- `/v1/models`

### `openai`

适合 OpenAI 兼容客户端，通常配合：

- `/v1/chat/completions`
- `/v1/responses`
- `/responses`

### `gemini`

适合 Gemini SDK / CLI，通常配合：

- `/v1beta/models`
- `/v1beta/models/{model}:generateContent`

### `antigravity`

这是一个特殊平台：

- 可以走 Antigravity 专用入口
- 也可能在部分场景参与混合调度
- 模型名经常需要以当前分组实际返回结果为准

### `sora`

Sora 独立走专用路径和配额规则，通常不和普通文本模型混着理解。

::: tip 如果你只是想先跑通
不要先比较一堆倍率和高级规则，先选一个状态正常、平台类型明确、自己知道怎么调用的分组即可。
:::

## 新用户如何选分组

### 目标是先跑通

优先选一个：

- 当前状态正常
- 自己清楚要接哪类客户端
- 能查到模型列表的分组

### 目标是长期稳定使用

再进一步比较：

- 倍率是否适合自己的预算
- 是否是订阅模式
- 是否有额度上限
- 是否有专属限制

## 一个实用建议

分组说明最终还是会变化，但**模型列表接口是实时答案**。所以任何时候你不确定某个分组当前能不能用某个模型，都先拿这把 Key 去查模型列表，不要只看历史截图或口口相传。
